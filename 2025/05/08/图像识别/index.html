导言

部分伪造检测（PFD）网络

1、伪造内容检测：确定内容真实性

2、伪造边界检测：检测真实片段和伪造片段的转换边界

自监督学习（SSL）模型：提高检测性能

分类

部分伪造检测（PFD）网络：从预测语句结果到帧级结果

时间伪造定位（TFL）网络：识别每个音频片段是真实的还是伪造的，还能确定伪造片段开始和结束的精确时间戳

从粗到细的检测，PFD定位到帧级，TFL具体识别伪造帧级的开始与结束

挑战

多片段篡改

提出新的两阶段

音频时态伪造检测和定位（TFDL）的框架，称为“从粗到细的提案细化框架”（CF-PRF）

第一阶段

帧级检测网络（FDN）

（与PFD不同的是，第一阶段用帧级检测网路FDN来学习）

为了放大小伪造片段造成的敏感差异，在差异感知特征学习（DAFL)）模块采用了对比学习法（CRL），引导不同的帧对在嵌入空间中表现出不相似性

设计了边界感知特征增强模块（BAFE），以引导边界上下文信息与时间差感知之间的交互，从而加强对多个伪造片段的检测

第二阶段

采用一个提案细化网路（PRN）来生成细粒度提案，PRN用于时态伪造定位

从FDN中提取粗粒度建议和区域特征，随后使用双预测头生成细粒度建议

任务定义

给定一段部分伪造的音频，我们旨在在帧级别检测部分伪造内容，并指出这些伪造片段的位置。

2.3.1 帧级别伪造检测

以输入音频$x$为例，它包含$T$帧。首先，通过一个设计好的编码器学习帧级别特征，记为$\hat{F}_x \in \mathbb{R}^{T \times D}$ 。然后，$\hat{F}_x$被解码为伪造概率分数$\hat{Y}_x = {\hat{y}_1, \hat{y}_2, \ldots, \hat{y}_T}$ 。此任务可被视为一个帧级别的分类挑战，使用标签$Y_x = {y_1, y_2, \ldots, y_T}$ ，其中$0$表示伪造帧，$1$表示真实帧。

2.3.2 时间伪造定位

每个音频$x$都关联着一组时间维度伪造片段的真实情况$G_x = {(s_m, d_m)}_{m = 1}^{M}$ ，其中$M$是音频中伪造片段的数量。对于第$m$个伪造片段，$s_m$和$d_m$分别是起始时间戳和持续时间。此任务需要为从FDN输出的每个粗粒度提议$\hat{P}_x^+ = {(\hat{s}_h^+, \hat{d}h^+)}{h = 1}^{H}$预测一组置信度分数$C_x = {c_1, c_2, \ldots, c_H}$ ，其中$H$是粗粒度提议的数量。然后，使用预测的回归偏移量$\hat{R}x = {(\hat{r}h^s, \hat{r}h^d)}{h = 1}^{H{pos}}$对正提议进行细化，其中$H{pos}$是正提议的数量，每个正提议都有一个置信度分数$c_h \geq \theta_p$ 。对于第$h$个提议，基于匹配的真实情况计算持续时间偏移量$r_h^d$（$r_h^d = \log(d_h / \hat{d}_h^+)$ ）和平移偏移量$r_h^s$（$r_h^s = (s_h - \hat{s}_h^+) / d_h$ ） 。

softmax函数解析

含义与原理

  • 名称拆解：softmax 由 “soft（软）” 和 “max（最大）” 组成 。普通的 max 函数是在多个值中取最大值，结果非黑即白 ；而 softmax 不是简单取最大值，它考虑所有输入值，以一种 “软” 的方式输出每个类别被选中的概率 。
  • 计算方式：对于输入向量x中的每个元素(x_i) ，其 softmax 输出值(y_i) 的计算公式为(y_i = \frac{e^{x_i}}{\sum_{j = 1}^{K} e^{x_j}}) 。其中K是x的维度，e是自然对数的底数 。该公式确保输出(y_i) 的值在((0, 1))之间，且所有(y_i) 的和为 1 ，满足概率分布要求 。例如，输入向量为((5, 7, 10)) ，经计算输出约为((0.006, 0.047, 0.946)) ，总和近似为 1 。

作用

  • 多分类概率转换：将神经网络输出的原始分数转换为概率分布，使每个类别的概率之和为 1 ，便于理解模型对不同类别的预测置信度 。例如在图像分类中，判断图片属于不同类别的概率 。
  • 平滑近似 arg max：arg max 操作返回最大值的索引，不可导 ；softmax 可视为 arg max 的平滑近似，具有良好数学性质，可用于基于梯度的优化方法 。

EER参数解析

EER 即 Equal Error Rate（等错误率 ），是在生物识别、音频 / 视频伪造检测等涉及二分类决策的领域中常用的性能评估指标 ，以下是详细介绍：

定义与计算原理

  • 两类错误：在二分类任务里，存在 “误报（False Acceptance，将负类误判为正类 ）” 和 “漏报（False Rejection，将正类误判为负类 ）” 两种错误。以音频伪造检测为例，误报是把真实音频误判成伪造音频，漏报是把伪造音频误判成真实音频 。
  • 调整阈值与绘制曲线：通过调整分类器的决策阈值（比如判断音频是否伪造的概率阈值 ），可以改变误报率（False Acceptance Rate，FAR ）和漏报率（False Rejection Rate，FRR ） 。不断改变阈值并记录对应的 FAR 和 FRR，以 FAR 为横坐标、FRR 为纵坐标绘制曲线 。
  • 等错误率确定：在这条曲线上，找到 FAR 和 FRR 相等的点，该点对应的错误率数值就是 EER 。它反映了在误报和漏报概率相等时系统的错误程度 。

作用

  • 性能衡量：作为一个单一数值指标，能简洁直观地评估系统性能。EER 越低，说明系统在平衡误报和漏报方面表现越好，分类决策越准确 。例如在指纹识别系统中，EER 低意味着既不会频繁把合法用户误拒，也不会轻易让非法用户通过 。
  • 系统比较：方便在不同检测系统、算法间进行性能对比。在选择音频伪造检测方案时，可依据 EER 判断哪个方案更可靠 。

mAP参数解析

mAP 即 Mean Average Precision（平均平均精度 ） ，是信息检索、目标检测等领域常用的评估指标，用于衡量模型对目标的检测和识别能力 ，以下是具体解析：

相关基础概念

  • Precision（精度）：在目标检测中，Precision 指的是模型预测为正样本（检测出目标 ）中，真正属于正样本（确实是目标 ）的比例 。公式为(Precision = \frac{TP}{TP + FP}) ，其中TP（True Positive）是真正例，即正确检测出的目标数量；FP（False Positive）是假正例，即误检的数量 。比如在图像的行人检测中，检测出 10 个行人，其中 8 个是真实行人，2 个是误检的，那么精度就是(\frac{8}{8 + 2} = 0.8) 。
  • Recall（召回率）：表示模型正确检测出的正样本（真正例 ）占所有实际正样本的比例 。公式为(Recall = \frac{TP}{TP + FN}) ，FN（False Negative）是假负例，即漏检的目标数量 。假设图像中有 15 个行人，检测出 8 个，召回率就是(\frac{8}{8 + (15 - 8)} = \frac{8}{15} \approx 0.53) 。
  • Average Precision（AP，平均精度）：针对每个类别，AP 是 Precision - Recall 曲线下的面积 。该曲线通过改变分类阈值，得到不同阈值下的 Precision 和 Recall 值并绘制而成。AP 综合考虑了模型在不同召回率水平下的精度表现，衡量了模型在该类别上的检测性能 。

mAP 计算及意义

  • 计算：在多类别目标检测任务中，分别计算每个类别的 AP，然后对所有类别的 AP 求平均值，得到的结果就是 mAP 。例如在一个包含行人、车辆、自行车三类目标的检测任务中，行人、车辆、自行车类别的 AP 分别为 0.7、0.6、0.5，那么 mAP 就是(\frac{0.7 + 0.6 + 0.5}{3} = 0.6) 。
  • 意义：mAP 能全面反映模型在多个类别目标检测上的综合性能 。mAP 值越高，说明模型在各类别目标的检测精度和召回率方面整体表现越好，即模型能够更准确地检测出各类目标，同时漏检和误检情况相对较少 。在评估不同目标检测算法时，mAP 是重要的参考指标之一 。
